# Project Charter

## Project Name

Recovery and Grief Website

## Project Type

Custom static website prototype and pre-launch build for a Massachusetts peer grief support program.

## Project Vision

Create a calm, trustworthy, recovery-affirming website that helps people in recovery quickly find grief support after a death caused by alcohol or other drugs.

## Problem Statement

The intended audience is navigating grief, recovery, stigma, and overwhelm at the same time. The website must reduce confusion, make support feel emotionally safe, and present next steps without sounding clinical or institutional.

## Primary Audience

- People in recovery in Massachusetts grieving a death caused by alcohol or other drugs

## Secondary Audiences

- Family or community members helping someone find support
- Recovery and grief-support partners making referrals
- Future collaborators using this website as a reusable program model

## Business And Program Goals

- Provide a clear public-facing entry point for the program
- Make direct support easy to find and easy to trust
- Present the partnership between SADOD and The Sun Will Rise clearly
- Build a foundation that can launch quickly and grow without a redesign restart
- Keep the website simple to maintain as a static site

## User Goals

- Understand within seconds that the site is for grieving people in recovery
- Know that support is peer-led and non-clinical
- Choose a next step without pressure
- Find contact, group, and resource options quickly

## In Scope

- Nine-page static website with About Us dropdown split
- Shared CSS and JavaScript
- Partner branding and page-level messaging
- Local preview workflow
- Launch and handoff documentation
- Hosting and maintenance planning documents

## Out Of Scope For This Phase

- Full CMS or admin dashboard
- Backend application development
- Stored form submissions inside the project itself
- Analytics implementation before launch decisions are approved
- Video embeds or media integrations before content is finalized

## Current Deliverable

An advanced-stage content and design build that is structurally complete, with remaining work concentrated on content approval, placeholder replacement, and production deployment.

## Success Criteria

- Visitors can understand the offer and next steps quickly
- The website feels emotionally safe, human, and credible
- Mobile and desktop layouts work cleanly
- Final contact pathways are obvious and low-friction
- The project can be handed off with clear documentation and low technical overhead

## Constraints

- Sensitive audience and tone requirements
- Final content is still awaiting approval in several places
- Contact workflow is not finalized
- The custom static-site direction is approved, but the final static host is not yet locked
- The project is still maintained as a lightweight file-based static site, so disciplined Git use and publish checks remain important

## Key Stakeholders

- Program owner / product sponsor
- Content approver / partner leadership
- Website designer-developer
- Future site maintainer or support contact

## Suggested Working Roles

- Product Owner: confirms scope, priorities, and approvals
- Project Manager: maintains backlog, roadmap, RAID log, and status
- Content Lead: approves wording, bios, resources, and trust signals
- Technical Lead: manages front-end implementation, QA, publishing, and ongoing site maintenance

## Working Assumptions

- A static site is the right delivery model for the current phase
- Squarespace may remain the domain registrar even if hosting moves elsewhere
- Contact, team, and resource details will be provided in a later content pass
- The current site structure is close enough to final to support focused refinement instead of a full IA restart

## Exit Criteria For Launch

- Final contact method is live and tested
- Final copy, bios, logos, and resources are approved
- Mobile and desktop QA are complete
- Publish checklist is complete
- Handoff and maintenance workflow are documented
