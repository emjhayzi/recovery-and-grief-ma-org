# Squarespace vs Native Website Build

Audited against the current repository state on April 12, 2026.

For this project, "native" means the current repo approach: standard HTML, CSS, and JavaScript deployed as a static site.

## Quick Comparison

| Area | Full Squarespace Build | Current Native Static Build |
| --- | --- | --- |
| Layout freedom | More constrained | Full control |
| Visual originality | More template-shaped | More tailored to the brand and tone |
| Editing model | Dashboard-first | Source-file workflow |
| Performance footprint | Platform overhead | Lightweight static output |
| Motion control | More limited | Fully controllable in source |
| Portability | More platform-tied | Easier to move |
| Current project fit | Requires re-platforming | Already matches the repo and deployment workflow |

## Project-Specific Take

Because the website is already implemented as a custom static build, the stronger path is to keep improving that build rather than translating it back into a more constrained template system.
